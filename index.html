🦞 OpenClaw Integration

OpenClaw uses ~/.openclaw/openclaw.json for config. Point the nvidia provider's baseUrl at this proxy — it will handle all key rotation transparently.

Step 1 — Set env var (proxy as fake key)
export NVIDIA_API_KEY=proxy   # proxy handles real keys — this value is ignored
Step 2 — Minimal openclaw.json (merge into existing)
{
  "models": {
    "mode": "merge",
    "providers": {
      "nvidia": {
        "baseUrl": "http://localhost:3000/v1",
        "api": "openai-completions"
      }
    }
  },
  "agents": {
    "defaults": {
      "model": {
        "primary": "nvidia/meta/llama-3.3-70b-instruct"
      },
      "models": {
        "nvidia/meta/llama-3.3-70b-instruct": {},
        "nvidia/meta/llama-4-maverick-17b-128e-instruct": {},
        "nvidia/deepseek-ai/deepseek-v3.2": {},
        "nvidia/deepseek-ai/deepseek-v4-flash": {},
        "nvidia/moonshotai/kimi-k2.5": {},
        "nvidia/nvidia/llama-3.1-nemotron-ultra-253b-v1": {},
        "nvidia/qwen/qwq-32b": {}
      }
    }
  }
}
⚠️ Model ref format: nvidia/<org>/<model-id>
e.g. nvidia/meta/llama-3.3-70b-instruct  ·  nvidia/moonshotai/kimi-k2.5
⚠️ Allowlist required: every model you use must appear in agents.defaults.models or OpenClaw will reject it.
⚠️ mode: "merge" is required so your custom provider is added to (not replaces) the bundled catalog.
Switch model via CLI:
openclaw models set nvidia/meta/llama-3.3-70b-instruct
Config file location:
~/.openclaw/openclaw.json

🤖 How routing works: When a request arrives for router/<slug>, the proxy classifies it into one of 9 task types — by keyword scoring (last message weighted 3×), long-text detection (>3000 words → Summarization), non-Latin script detection (Arabic/CJK/etc → Translation), or image presence (→ Multimodal). It then transparently swaps the model ID before forwarding to NVIDIA.
